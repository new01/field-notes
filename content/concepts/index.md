These are the mental models that matter most when building with OpenClaw. Not feature documentation — the underlying patterns that explain why things work the way they do.

Start with [[concepts/brains-and-muscles|Brains and Muscles]]. It's the frame everything else fits into.

## All concepts

- **[[concepts/brains-and-muscles|Brains and Muscles]]** — the two-layer model of every agent system: reasoning vs. action
- **[[concepts/agent-memory|Agent Memory]]** — how to give your agent continuity across sessions so value compounds over time
- **[[concepts/prompt-file-governance|Prompt File Governance]]** — every line in auto-loaded files costs tokens on every request; treat them like a stylesheet
- **[[concepts/agent-teams|Agent Teams]]** — spawning multiple specialized subagents to handle parallel workstreams
- **[[concepts/self-improvement-system|Self-Improvement System]]** — a feedback loop where the agent tracks its own errors and proposes fixes
